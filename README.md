# 💻 Изучение языка программирования C++

> Структурированный путь изучения C++: от фундаментальных принципов работы с памятью и типами до низкоуровневой оптимизации, RAII и STL. Курс построен на "Programming: Principles and Practice Using C++" Бьярне Страуструпа (2-е издание) с постоянной сверкой на 3-е издание (C++20/23/26) и внешние источники — C++ Core Guidelines, Скотт Мейерс, Metanit.

Репозиторий для практических задач, экспериментов и глубокого изучения системного программирования на C++ от самых основ до продвинутого уровня. Каждая глава фиксируется отдельным Git-чекпоинтом

```jsonc
    "-Wextra",

    "-o",
    "${fileDirname}\\${fileBasenameNoExtension}.exe",
    "-std=c++26"
]
```

## 🏆 Избранные проекты и алгоритмы

### 🎮 Камень, Ножницы, Бумага (4.10)

- 📄 **Файл:** [Rock_Paper_Scissors](Phase-1-Basics/Ch-04-Computation/Ex10_Rock_Paper_Scissors.cpp)
- 🔨 **Стек:** `std::vector`, оператор `switch-case`, псевдогенерация ходов компьютера.
- ⚙️ **Инженерные детали:**
  - **Валидация ввода как отдельный контур.**
    > Ввод читается и валидируется в цикле `while` до получения корректного значения; отдельно обрабатывается состояние ошибки потока (`cin.fail()`) при нечисловом вводе, с последующим `cin.clear()` + `cin.ignore(numeric_limits<streamsize>::max(), '\n')` для полной очистки буфера — без этого поток необратимо "зависает" в состоянии ошибки.
  - **Логика сравнения через модульную арифметику.**
    > Результат раунда вычисляется одной формулой `(player - computer + 3) % 3` вместо перебора всех 9 комбинаций через `if/else` — компактнее и меньше точек отказа.
  - **Разделение ответственности.**
    > Вывод меню, чтение/валидация ответа и основной игровой цикл — три независимые функции, каждая с одной зоной ответственности.

### 🔍 Поиск простых чисел (4.11 - 4.15)

- 📄 **Файл:** [Prime numbers](Phase-1-Basics/Ch-04-Computation/Ex11-15_PrimeNums.cpp)
- ✏️ **Теоретическая заметка:** [Prime_Numbers.md](Phase-1-Basics/Ch-04-Computation/primes_notes.md)
- ⚙️ **Инженерные детали:**
  - **Математическая элегантность: Теорема Россера.**
    > Вместо того чтобы гадать или динамически расширять вектор (что привело бы к дорогой переаллокации памяти), программа за $O(1)$ вычисляет гарантированный верхний предел значений. Выделяется ровно один массив нужного размера, отрабатывает быстрый классический алгоритм, и мы гарантированно получаем не менее $N$ простых чисел.
  - **Защита от переполнения (Overflow Prevention).**
    > Приведение одного из множителей к (long long) форсирует выполнение умножения в 64-битном пространстве, полностью предотвращая переполнение.
  - **Вспомогательная функция расчета времени выполнения**
    > `milliseconds howLong(function<void()> what)`
  - **Написан теоретический разбор математической основы простых чисел.**

## ⚙️ Инструкция по сборке

Требуется установленный GCC (рекомендуется через **MSYS2 UCRT64**) с поддержкой `-std=c++26`.

**Компиляция одного файла из терминала:**

```bash
g++ -std=c++26 -Wall -Wextra -g -fdiagnostics-color=always main.cpp -o main.exe
```

**Запуск (Windows / PowerShell):**

```powershell
.\main.exe
```

**Проверка версии компилятора:**

```bash
g++ --version
```

> В VS Code сборка запускается автоматически для активного файла через `Ctrl+Shift+B` — конфигурация уже прописана в `.vscode/tasks.json`.
